ADR-008 Per-Tenant Bundles: State Machine Revision

Comparing current design vs revised design with critical fixes

Current Design (from existing spec)
Active → Parked → Cold
        ┌────────────┐
   write│            │ idle timeout
┌──────►│   PARKED   ├───────►┌────────┐
│      │            │        │  COLD  │
│      └────────────┘        └────────┘
│            ▲                     │
│            │ write                 │ read (lazy warmup)
│            │                      │
│     ┌──────┴──────┐               │
│     │   ACTIVE    │◄──────────────┘
└─────┤             │
      │ WAL open    │
      │ memtable hot│
      └─────────────┘

      Problems:
      • No error state — WAL corruption = panic or silent failure
      • No lock ordering defined — eviction can deadlock with writes
      • Cold→Active rebuilds locator (unbounded memory for 10K tenants)
Revised Design
Active → Parked → Cold → Unhealthy
        ┌────────────┐     failure     ┌────────────┐
   write│            ├──────────────►│  UNHEALTHY │
┌──────►│   PARKED   │   (retry)     │   (err)    │
│      │            │◄──────────────┘            │
│      └─────┬──────┘        backoff              │
│            │ idle timeout                       │
│            ▼                                    │
│      ┌──────────┐                               │
│      │   COLD   │                               │
│      │(no RAM)  │                               │
│      └─────┬────┘                               │
│            │ read                               │
│            │ (lazy)                             │
│            ▼                                    │
│      ┌──────────┐◄───────────────────────────────┘
│      │  ACTIVE  │
└──────┤          │
       │ WAL open │
       │ locator  │
       │ memtable │
       └──────────┘

      Critical Fixes:
      • UNHEALTHY state for WAL corruption / disk errors
      • Lock ordering: tenantMapMu → bundle.mu (never reverse)
      • Cold drops locator from RAM (rebuild on warmup)
      • Eviction removes from map BEFORE locking bundle
A

Keep Current Design

Implement Approach C as originally specified. Add error handling later if needed.

B

Use Revised Design

Add Unhealthy state, define lock ordering, and drop Cold locator before implementing.

C

Hybrid — Incremental

Implement basic per-tenant bundles first, add Unhealthy state and memory optimizations in follow-up.
